# DURO

A design system engineered for durability. Zero runtime overhead, future-proof patterns, and a composable architecture that survives framework churn.

---

## Technical Architecture

<Stack>
  <Card>
    <CardHeader>
      <Badge variant="primary">Zero Runtime</Badge>
      <CardTitle style={{ marginTop: '1rem' }}>Build-Time CSS Generation</CardTitle>
    </CardHeader>
    <CardContent>
      <Paragraph>
        Duro's theme system runs at build time, converting TypeScript theme objects into CSS custom properties.
        A Node.js script reads your theme configuration and generates a static duro.css file.
        No JavaScript theme context, no runtime overhead, perfect for server-side rendering.
      </Paragraph>
    </CardContent>
  </Card>

  <Card>
    <CardHeader>
      <Badge variant="success">Modern</Badge>
      <CardTitle style={{ marginTop: '1rem' }}>Container Queries</CardTitle>
    </CardHeader>
    <CardContent>
      <Paragraph>
        Components respond to their container size, not the viewport. This enables true component-driven responsive design.
        A Grid behaves the same whether it's in a sidebar or full-width, because it queries its own container.
      </Paragraph>
    </CardContent>
  </Card>

  <Card>
    <CardHeader>
      <Badge variant="outline">Composable</Badge>
      <CardTitle style={{ marginTop: '1rem' }}>Layout Component System</CardTitle>
    </CardHeader>
    <CardContent>
      <Paragraph>
        Duro provides layout primitives (Stack, Grid, Cluster, Sidebar) that compose together to build interfaces.
        Each component has variants defined in the theme that generate corresponding CSS classes at build time.
      </Paragraph>
    </CardContent>
  </Card>
</Stack>

## Design Principles

<Stack>
  <Card>
    <CardHeader>
      <CardTitle>HUD Tech Aesthetic</CardTitle>
    </CardHeader>
    <CardContent>
      <Paragraph>
        Monospace typography, monochromatic colors, and sharp geometric shapes.
        Corner brackets, dashed borders, and terminal-style interfaces define Duro's HUD character.
      </Paragraph>
    </CardContent>
  </Card>

  <Card>
    <CardHeader>
      <CardTitle>Framework Agnostic</CardTitle>
    </CardHeader>
    <CardContent>
      <Paragraph>
        Built with React but designed to outlast it. CSS custom properties mean your theme survives framework migrations.
        The component layer is thin - most logic lives in composable hooks and pure functions.
      </Paragraph>
    </CardContent>
  </Card>

  <Card>
    <CardHeader>
      <CardTitle>Progressive Enhancement</CardTitle>
    </CardHeader>
    <CardContent>
      <Paragraph>
        Semantic HTML first. Components work without JavaScript, then enhance with interactivity.
        Container queries degrade gracefully. Baseline functionality doesn't require runtime.
      </Paragraph>
    </CardContent>
  </Card>
</Stack>

## Get Started

```bash
pnpm add @caryyon/duro
```

Then import the CSS and start using components:

```tsx
import '@caryyon/duro/dist/duro.css'
import { Button, Card, Stack } from '@caryyon/duro'
```
